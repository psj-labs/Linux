# Hard Disk, Partition, File System, Mount 

---

## Hard Disk (하드디스크)

- 하드디스크는 데이터를 물리적으로 저장하는 장치입니다.
- 운영체제는 하드디스크를 직접 사용하지 않고, 내부를 나누어 관리합니다.
- Linux에서는 디스크가 디바이스 파일 형태로 인식됩니다.

- `예시`
```text
- /dev/sda
- /dev/nvme0n1
```
- 디스크 자체에는 파일을 바로 저장할 수 없으며, 파티션과 파일 시스템 구성이 필요합니다.

---

## Partition (파티션)

- 파티션은 하나의 물리적 디스크를 여러 개의 논리적인 영역으로 나눈 것입니다.
- 각 파티션은 독립적으로 파일 시스템을 가질 수 있습니다.

### Primary Partition
- 실제 데이터를 저장할 수 있는 기본 파티션입니다.
- 하나의 디스크에 최대 4개까지 생성할 수 있습니다.
- 운영체제가 직접 사용할 수 있습니다.

### Extended Partition
- Logical Partition을 만들기 위한 컨테이너 역할의 파티션입니다.
- 디스크당 1개만 생성할 수 있습니다.
- 직접 데이터를 저장하지는 않습니다.

### Logical Partition
- Extended Partition 내부에 생성됩니다.
- 사용 방식은 Primary Partition과 동일합니다.
- 파티션 개수 제한을 넘기기 위해 사용됩니다.

- `Primary Partition` + `Extended Partition`은 최대 4개까지 생성 가능합니다.

---

## File System (파일 시스템)

- 파일 시스템은 디스크나 파티션 위에 데이터를 저장하고 관리하는 규칙과 구조입니다.
- 파일 이름, 디렉터리 구조, 권한, 메타데이터를 관리합니다.
- 파일 시스템이 없으면 디스크에 데이터를 저장할 수 없습니다.

- `대표적인 Linux 파일 시스템`
```text
- ext4
- xfs
- btrfs
```
- `예시`
```text
- mkfs.xfs /dev/sda1
- mkfs.ext4 /dev/nvme0n1p1
```
- 파일 시스템을 생성하는 과정은 디스크를 실제로 사용 가능한 상태로 만드는 단계입니다.

---

## Mount (마운트)

- 마운트는 파일 시스템을 특정 디렉터리에 연결하여 접근 가능하게 만드는 과정입니다.
- Linux에서는 모든 파일이 하나의 루트(/) 트리 아래에 연결됩니다.

- `예시`
```text
- /dev/sda1 -> /home
- /dev/nvme0n1p1 -> /data
```
- 마운트가 되지 않은 파일 시스템은 존재하더라도 접근할 수 없습니다.
- 마운트 해제(umount)는 파일 시스템을 안전하게 분리하는 과정입니다.

4. 파일 시스템을 디렉터리에 마운트
5. 파일 생성 및 사용

---

## 정리

- 하드디스크는 물리적 저장 장치입니다.
- 파티션은 디스크를 나눈 논리적 영역입니다.
- 파일 시스템은 데이터를 저장하기 위한 구조입니다.
- 마운트는 파일 시스템을 디렉터리에 연결하는 과정입니다.
- 이 네 단계가 모두 완료되어야 디스크를 정상적으로 사용할 수 있습니다.
