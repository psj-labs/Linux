# Routing 정보 추가 & ARP 엔트리 수동 추가

## 개요

- 본 문서는 Linux 환경에서 라우팅 테이블과 ARP 테이블을 확인·수정하는 과정을 정리한 학습 문서입니다.
- `ip route`, `ip neighbor` 명령어를 중심으로 라우팅·ARP의 동작을 확인합니다.
- 또한 `/etc/hosts`, `/etc/sysconfig/network-scripts/ifcfg-*` 파일을 이용한 **영구 설정 방식**을 함께 정리합니다.

---

## 라우팅 테이블 확인

- 라우팅 테이블 확인 명령어는 다음과 같습니다.
```text
ip route show  
또는 축약형  
ip r
```
![29](/Network%20Setting%20-%20Part%201/imgs/29.png)

- 필자는 현재 2개의 NIC를 사용 중이며 라우팅 테이블을 확인합니다.
- 첫 번째와 두 번째 줄에는 각 인터페이스에 해당하는 디폴트 게이트웨이 정보가 출력됩니다.
- 각 라우트는 “어떤 네트워크로 가기 위해 어떤 인터페이스를 사용하며, 출발지 IP를 무엇으로 사용할지”를 의미합니다.

---

## Default Route 해석
```text
default via 192.168.11.1 dev ens224 proto static metric 101
```
- 의미  
  목적지를 모르는 모든 트래픽은  
  192.168.11.1로, ens224 인터페이스를 통해 전송됩니다.

---

## 특정 네트워크 Route 해석
```text
192.168.10.0/24 dev ens160 proto kernel scope link src 192.168.10.147 metric 100
```
- 의미  
  192.168.10.0/24 네트워크로 가는 패킷은  
  ens160 인터페이스를 사용하며  
  출발지 IP는 192.168.10.147을 사용합니다.

---

## ARP 테이블 확인

- ARP(Neighbor) 테이블 확인 명령어는 다음과 같습니다.
```text
ip neighbor show  
또는 축약형  
ip n
```
![30](/Network%20Setting%20-%20Part%201/imgs/30.png)

---

## ARP 엔트리 해석
```text
192.168.10.146 dev ens160 lladdr 3c:7c:3f:7d:f0:c4 REACHABLE
```
- 의미  
  192.168.10.146은 ens160 인터페이스에 연결된 장비이며  
  MAC 주소는 3c:7c:3f:7d:f0:c4이고  
  현재 통신 가능한 상태입니다.
```text
192.168.10.1 dev ens160 lladdr 58:86:94:49:b7:3c REACHABLE
```
- 의미  
  해당 MAC 주소는  
  192.168.10.0/24 네트워크의 관문 역할을 하는  
  라우터의 NIC MAC 주소입니다.

---

## ARP 명령어 정리

| 구분 | 명령어 |
|---|---|
| 전체 ARP 테이블 확인 | ip neighbor show / ip n |
| 특정 인터페이스 ARP 확인 | ip n show dev [NIC] |
| 특정 IP ARP 확인 | ip n show [IP] |
| ARP 엔트리 수동 추가 | ip n add [IP] lladdr [MAC] dev [NIC] |
| 특정 ARP 엔트리 삭제 | ip n del [IP] dev [NIC] |
| ARP 테이블 전체 초기화 | ip n flush all |
| 특정 인터페이스 ARP 초기화 | ip n flush dev [NIC] |

---

## ARP 엔트리 수동 추가

- 다음과 같이 ARP 정보를 수동으로 추가합니다.
```text
ip n add 192.168.11.200 lladdr 00:1c:2d:3e:4f:05 dev ens224
```
![31](/Network%20Setting%20-%20Part%201/imgs/31.png)

- `ip n`으로 다시 확인 시  
  수동으로 추가한 ARP 정보가 등록된 것을 확인할 수 있습니다.

---

## ARP 엔트리 삭제

- 다음 명령어로 ARP 엔트리를 삭제합니다.
```text
ip n del 192.168.11.200 dev ens224
```
![32](/Network%20Setting%20-%20Part%201/imgs/32.png)

- `ip n`으로 재확인 시  
  해당 ARP 엔트리가 정상적으로 삭제된 것을 확인할 수 있습니다.

## 라우팅 테이블 재확인

- 현재 라우팅 테이블은 다음 명령어로 확인할 수 있습니다.
```text

```
![33](/Network%20Setting%20-%20Part%201/imgs/33.png)

- net-tools가 설치된 경우 다음 명령어도 사용 가능합니다.
```text
route
```
![34](/Network%20Setting%20-%20Part%201/imgs/34.png)

---

## 라우팅 테이블 수동 변경 (임시)

- 라우트 추가 및 삭제 명령 형식은 다음과 같습니다.
```text
ip route add/del [target ip/mask] via [gateway ip] dev [NIC]
```
- 해당 설정은 **임시 설정**입니다.

---

## Default Gateway 삭제 및 추가

- 기존 Default Gateway 삭제
```text
ip route del default via 192.168.10.1
```
![35](/Network%20Setting%20-%20Part%201/imgs/35.png)

- 새로운 Default Gateway 추가
```text
ip route add default via 192.168.10.1 dev ens160
```
![36](/Network%20Setting%20-%20Part%201/imgs/36.png)

- `ip r`로 확인 시 정상적으로 추가된 것을 확인할 수 있습니다.

## /etc/hosts 파일

- `/etc/hosts` 파일은 호스트명과 IP 주소를 매핑하는 파일입니다.
- 각 레코드는 다음 형식으로 정의됩니다.
```text
[ip] [호스트명(도메인)] [별명]
```
- Name Resolution 순서는 `/etc/host.conf` 파일에서 정의됩니다.

---

## /etc/host.conf 역할

- name resolution 순서를 정의하는 파일입니다.
- 주요 항목은 다음과 같습니다.
```text
hosts : /etc/hosts 파일 검사  
bind  : /etc/resolv.conf에 지정된 DNS 서버 질의
```

---

## 정리

- `ip route`, `ip neighbor` 명령어를 통해  
  라우팅 테이블과 ARP 테이블을 확인하고 수정할 수 있습니다.
- `ip route add/del`, `ip n add/del`은 **임시 설정**입니다.
- `/etc/hosts`, `/etc/sysconfig/network-scripts/ifcfg-###` 수정은  
  **영구 설정**입니다.
- 실무 환경에서는 임시 설정과 영구 설정의 차이를 명확히 구분해야 합니다.
